1. Filtro de fraude

El filtro de Fraude le permite establecer reglas en el comercio electrónico mantener, bloquear o pasar transacciones. También puede proporcionar información adicional en la solicitud con la que puede establecer reglas. Esto incluye la dirección IP del cliente, los detalles de facturación y envío y los códigos específicos del producto.

Solo es posible establecer el modo Filtro de fraude en PASIVO o APAGADO para transacciones individuales en la solicitud. Si desea configurarlo como ACTIVO, debe configurarlo en el portal de comercio electrónico y se aplicará a todas las transacciones.

1.1 Código de ejemplo
using Microsoft.AspNetCore.Mvc;
using GlobalPayments.Api;
using GlobalPayments.Api.Entities;
using GlobalPayments.Api.PaymentMethods;

namespace AddonPayments.Controllers {
[ApiController]
[Route("[Controller]")]

public class FraudApiController: ControllerBase {
[HttpPost]
public IActionResult FraudApi() {

// Configuramos los datos de nuestro comercio. Los datos son proporcionados por el equipo de Addon Payments,
// en caso de duda debe llamar al 914 353 028 e indicar su número de comercio o Merchant ID
ServicesContainer.ConfigureService(new GatewayConfig {
MerchantId = "addonnettest",
AccountId = "api",
SharedSecret = "secret",
ServiceUrl = "https://remote.sandbox.addonpayments.com/remote"
});

// Creamos el objeto de la tarjeta
var card = new CreditCardData {
Number = "4263970000005262",
ExpMonth = 12,
ExpYear = 2025,
Cvn = "123",
CardHolderName = "Nombre del titular"
};

// Creamos el objeto billingAddress
var billingAddress = new Address {
Country = "ES",
PostalCode = "03201"
};

// Creamos el objeto shippingAddress
var shippingAddress = new Address {
Country = "ES",
PostalCode = "03201"
};

// Lanzamos la petición al servidor de Addon Payments
try {
Transaction response = card.Charge(19.95m)
.WithCurrency("EUR")
.WithAddress(billingAddress, AddressType.Billing)
.WithAddress(shippingAddress, AddressType.Shipping)
.WithProductId("1QSD67KLDUI23")
.WithClientTransactionId("VARREF")
.WithCustomerId("16547982359")
.WithCustomerIpAddress("123.123.123.123")
.Execute();

// Obtenemos la respuesta y la mostramos
var result = response.ResponseCode; // 00 == Success
var message = response.ResponseMessage; // [ test system ] AUTHORISED
var orderId = response.OrderId; // Order ID de la transacción
var authCode = response.AuthorizationCode; // Código de autorización
var paymentsReference = response.TransactionId; // Identificador de transacción (PASREF)
var schemeReferenceData = response.SchemeId; // MMC0F00YE4000000715

// Devolvemos la respuesta asociando los parámetros recibidos por el servidor a una variable
// para más información puede ver el controlador "AuthorizationApiController.cs"
return Ok(variable);

// Aconsejamos guardar los datos de la operacion en su base de datos en caso de querer gestionar de forma interna sus transacciones

} catch (ApiException exce) {
// Añada aquí su tratamiento de errores
return Ok(exce);
}
}
}
}

2. Descripciones

2.1 Configuración del terminal

Debemos introducir los datos de configuración que se facilitan en la llamada de cualificación del soporte técnico de Addon Payments.

  • Merchant_ID = Identificador del comercio.

  • SubAccount = Subcuenta faciltiada en la llamada de cualificación (en caso de que se envíe la operación sin subcuenta, se usará la establecida por defecto en la creación de su terminal).

  • Shared Secret = Contraseña de seguridad del terminal.
--> En caso de que le falte algún dato de los mencionados arriba, debe contactar con su soporte técnico de Addon Payments llamando al teléfono 914 353 028 (opción 2) o enviando un email al correo electrónico soporteaddonpayments.com.
ServicesContainer.ConfigureService(new GatewayConfig {
MerchantId = "addonnettest", // Identificador del comercio
AccountId = "api", // Subcuenta asociada a su identificador
SharedSecret = "secret", // Contraseña del terminal
ServiceUrl = "https://remote.sandbox.addonpayments.com/remote" // Se debe emplear una de las siguientes URL's:
// Sandbox URL para el entorno de pruebas - https://remote.sandbox.addonpayments.com/remote.
// Live URL para el entorno de producción - https://remote.addonpayments.com/remote.
});
2.2 Datos del cliente

La información que debemos enviar al servidor es la siguiente:

  • Creamos un nuevo objeto tarjeta llamado "new CreditCardData":
    • new CreditCardData

  • Indicamos cuál es el número de la tarjeta a la cual vamos a realizarle el cargo: "4263970000005262"
    • Number = "4263970000005262"

  • Indicamos el mes de expiración:
    • ExpMonth = "12"

  • Indicamos la fecha de expiración:
    • ExpYear = "2025"

  • Indicamos el código de seguridad de la tarjeta:
    • Cvn = "123"

  • Nombre del titular de la tarjeta:
    • CardHolderName = "Nombre del cliente"

  • Creamos un nuevo objeto para la dirección de facturación llamando a la clase "new Address":
    • billingAddress = new Address

  • Indicamos el el país de facturación y código postal del cliente:
    • Country = "ES"
    • PostalCode = "03201"

  • Creamos un nuevo objeto para la dirección de envío llamando a la clase "new Address":
    • shippingAddress = new Address

  • Indicamos el el país de envío y código postal del cliente:
      • Country = "ES"
    • PostalCode = "03201"
// Creamos el objeto billingAddress
var billingAddress = new Address {
Country = "ES",
PostalCode = "03201"
};

// Creamos el objeto shippingAddress
var shippingAddress = new Address {
Country = "ES",
PostalCode = "03201"
};
2.3 Envío de la operación

Para enviar la operación debemos emplear el modelo de excepciones "try - catch". -->Para obtener más información sobre la excepción de "try - catch" puede ver la documentación oficial de .NET

  • Cargamos el importe en la tarjeta:
    • response = card.Charge(19.95m);

  • Indicamos la moneda con la que vamos a realizar la operación:
    • .WithCurrency("EUR")

  • Añadimos la dirección de facturación:
    • .WithAddress(billingAddress, AddressType.Billing)

  • Añadimos la dirección de envío:
    • .WithAddress(shippingAddress, AddressType.Shipping)

  • Añadimos el identificador del producto:
    • .WwithProductId("SKU")

  • Añadimos la dirección de envío:
    • .WithClientTransactionId("VARREF")

  • Añadimos el identificador del cliente:
    • .WithCustomerId("16547982359")

  • Añadimos la dirección IP del cliente:
    • .WithCustomerIpAddress("123.123.123.123")

  • Ejecutamos la operación contra los servidores de Addon Payments:
    • .Execute();
try {
