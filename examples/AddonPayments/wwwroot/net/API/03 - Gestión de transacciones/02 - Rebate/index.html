1. Rebate (Devolver transacción)

Un Rebate es una devolución a una autorización existente capturada con éxito. Para que se acepte la solicitud, debe proporcionar el ID de pedido original, la Referencia de pagos (pasref) y el Código de autenticación de la respuesta de autorización. Las transacciones se pueden reembolsar hasta el 115% del importe de la autorización original y la moneda original también se debe enviar en la solicitud. Para procesar un reembolso, debe proporcionar la contraseña de Rebate que puede establecer con un miembro de nuestro equipo de soporte Addon Payments. Esta contraseña debe cifrarse utilizando el algoritmo SHA-1 antes de agregarse a la solicitud.

Es posible procesar múltiples reembolsos contra una transacción. Sin embargo, esto no está habilitado de forma predeterminada en su cuenta de comercio. En caso de querer activar esta operativa, deben contactar con el equipo de soporte para su activación. --> Puede contactar con nosotros llamando al teléfono 914 353 028 opción 2 o enviándonos un email a soporte@addonpayments.com.

using Microsoft.AspNetCore.Mvc;
using GlobalPayments.Api;
using GlobalPayments.Api.Entities;

namespace AddonPayments.Controllers {
[ApiController]
[Route("[Controller]")]

public class SettleApiController: ControllerBase {
[HttpPost]
public IActionResult SettleApi() {

// Configuramos los datos de nuestro comercio. Los datos son proporcionados por el equipo de Addon Payments,
// en caso de duda debe llamar al 914 353 028 e indicar su número de comercio o Merchant ID
ServicesContainer.ConfigureService(new GatewayConfig {
MerchantId = "addonnettest",
AccountId = "api",
SharedSecret = "secret",
RebatePassword = "rebate",
ServiceUrl = "https://remote.sandbox.addonpayments.com/remote"
});

// Identificador de la transacción que queremos capturar
var orderId = "15-50_12_12_2019-SDK2-0-0-UdeVsmgS3UeePLDfBfnfSQ";

// Referencia de la transacción original
var paymentsReference = "15761622418795589";

// Código de autorización
var authCode = "12345";

// Creamos el objeto de transaction
var transaction = Transaction.FromId(paymentsReference, orderId);
transaction.AuthorizationCode = authCode;

// Lanzamos la petición al servidor de Addon Payments
try {
Transaction response = transaction.Refund(19.95m)
.WithCurrency("EUR")
.Execute();

// Obtenemos la respuesta y la mostramos
var result = response.ResponseCode; // 00 == Success
var message = response.ResponseMessage; // [ test system ] AUTHORISED
var orderId = response.OrderId; // Order ID de la transacción
var authCode = response.AuthorizationCode; // Código de autorización
var paymentsReference = response.TransactionId; // Identificador de transacción (PASREF)
var schemeReferenceData = response.SchemeId; // MMC0F00YE4000000715

// Devolvemos la respuesta asociando los parámetros recibidos por el servidor a una variable
// para más información puede ver el controlador "AuthorizationApiController.cs"
return Ok(variable);

// Aconsejamos guardar los datos de la operacion en su base de datos en caso de querer gestionar de forma interna sus transacciones

} catch (ApiException exce) {
// Añada aquí su tratamiento de errores
return Ok(exce);
}
}
}
}

2. Descripciones

2.1 Configuración del terminal

Debemos introducir los datos de configuración que se facilitan en la llamada de cualificación del soporte técnico de Addon Payments.

  • Merchant_ID = Identificador del comercio.
  • SubAccount = Subcuenta faciltiada en la llamada de cualificación (en caso de que se envíe la operación sin subcuenta, se usará la establecida por defecto en la creación de su terminal).
  • Shared Secret = Contraseña de seguridad del terminal.
  • Rebate Password = Contraseña de Rebate.
--> En caso de que le falte algún dato de los mencionados arriba, debe contactar con su soporte técnico de Addon Payments llamando al teléfono 914 353 028 (opción 2) o enviando un email al correo electrónico soporteaddonpayments.com.
ServicesContainer.ConfigureService(new GatewayConfig {
MerchantId = "addonnettest", // Identificador del comercio
AccountId = "api", // Subcuenta asociada a su identificador
SharedSecret = "secret", // Contraseña del terminal
ServiceUrl = "https://remote.sandbox.addonpayments.com/remote" // Se debe emplear una de las siguientes URL's:
// Sandbox URL para el entorno de pruebas - https://remote.sandbox.addonpayments.com/remote.
// Live URL para el entorno de producción - https://remote.addonpayments.com/remote.
});
2.2 Datos de la transacción original

Para realizar una devolución, debemos indicar los siguientes datos:

--> Estos datos se facilitan en la respuesta de cada operación.

  • orderId = Identificador de la transacción original.

  • paymentsReference = Referencia de la operación preautorizada. Se obtiene en la respuesta de la preautorización.

  • authCode = Código de autorización.

  • Creamos el objeto "transaction" con las referencias que hemos asociado anteriormente:
    • transaction = Transaction.FromId(paymentsReference, orderId );
    • transaction.AuthorizationCode = authCode;
// Identificador de la transacción que queremos capturar
var orderId = "15-50_12_12_2019-SDK2-0-0-UdeVsmgS3UeePLDfBfnfSQ";

// Referencia de la transacción original
var paymentsReference = "15761622418795589";

// Código de autorización
var authCode = "12345";

// Creamos el objeto de transaction
var transaction = Transaction.FromId(paymentsReference, orderId);
transaction.AuthorizationCode = authCode;
2.3 Envío de la petición Rebate

Para enviar la solicitud a los servidores de Addon Payments, debemos emplear el modelo de excepciones "try - catch". -->Para obtener más información sobre la excepción de "try - catch" puede ver la documentación oficial de .NET

  • Indicamos el importe que queremos devolver de la transacción original:
    • response = transaction .Refund(19.95m)

  • Debemos indicar la moneda de liquidación de la operación original:
    • .WithCurrency("EUR")

  • Ejecutamos la operación:
    • .Execute();

try {
Transaction response = transaction.Refund(19.95m)
.WithCurrency("EUR")
.Execute();

// Obtenemos la respuesta y la mostramos
var result = response.ResponseCode; // 00 == Success
var message = response.ResponseMessage; // [ test system ] AUTHORISED
var orderId = response.OrderId; // Order ID de la transacción
var authCode = response.AuthorizationCode; // Código de autorización

// Devolvemos la respuesta asociando los parámetros recibidos por el servidor a una variable
// para más información puede ver el controlador "AuthorizationApiController.cs"
return Ok(variable);

// Aconsejamos guardar los datos de la operacion en su base de datos en caso de querer gestionar de forma interna sus transacciones

} catch (ApiException exce) {
// Añada aquí su tratamiento de errores
return Ok(exce);
}

Para la respuesta realizamos un return Ok() de "exce".

Con esta función obtenemos el motivo de la denegación del servidor. Para ver todos los errores de denegación actualizados, puede comprobar nuestra documentación de Addon Payments.
