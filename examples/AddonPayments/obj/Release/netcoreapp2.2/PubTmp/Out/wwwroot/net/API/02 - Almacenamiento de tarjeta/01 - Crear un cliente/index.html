1. Crear cliente

Para almacenar una tarjeta, lo primero que tenemos que hacer es configurar una referencia de cliente (Payer). También puede elegir almacenar la dirección y los datos de contacto junto a ella.

1.1 Código de ejemplo
using Microsoft.AspNetCore.Mvc;
using GlobalPayments.Api;
using GlobalPayments.Api.Entities;
using System;

namespace AddonPayments.Controllers {
[ApiController]
[Route("[Controller]")]

public class CustomerApiController: ControllerBase {
[HttpPost]
public IActionResult CustomerCreate() {

// Configuramos los datos de nuestro comercio. Los datos son proporcionados por el equipo de Addon Payments,
// en caso de duda debe llamar al 914 353 028 e indicar su número de comercio o Merchant ID
ServicesContainer.ConfigureService(new GatewayConfig {
MerchantId = "addonnettest",
AccountId = "api",
SharedSecret = "secret",
ServiceUrl = "https://remote.sandbox.addonpayments.com/remote"
});

// Creamos el objeto del cliente
var customer = new Customer {
Key = Guid.NewGuid().ToString(),
Title = "Señor/Señora",
FirstName = "Nombre",
LastName = "Apellidos",
DateOfBirth = "Cumpleaños",
CustomerPassword = "Password",
Email = "Email",
HomePhone = "Teléfono",
DeviceFingerPrint = "Huella digital",
Fax = "Fax",
WorkPhone = "Teléfono del trabajo",
Comments = "Comentarios",
Company = "Compañía",
Address = new Address {
StreetAddress1 = "Dirección 1",
StreetAddress2 = "Dirección 2",
StreetAddress3 = "Dirección 3",
City = "Ciudad",
Province = "Provincia",
PostalCode = "Código postal",
Country = "ISO del país (ES, GB, US...)",
}
};

// Lanzamos la petición al servidor de Addon Payments
try {
var response = customer.Create();
var payerRef = response.Key;

// Devolvemos la respuesta asociando los parámetros recibidos por el servidor a una variable
// para más información puede ver el controlador "AuthorizationApiController.cs"
return Ok(response.Key);

// Aconsejamos guardar los datos de la operacion en su base de datos en caso de querer gestionar de forma interna sus transacciones

} catch (ApiException exce) {
// Añada aquí su tratamiento de errores
return Ok(exce);
}
}
}
}

2. Descripciones

2.1 Configuración del terminal

Debemos introducir los datos de configuración que se facilitan en la llamada de cualificación del soporte técnico de Addon Payments.

  • Merchant_ID = Identificador del comercio.
  • SubAccount = Subcuenta faciltiada en la llamada de cualificación (en caso de que se envíe la operación sin subcuenta, se usará la establecida por defecto en la creación de su terminal).
  • Shared Secret = Contraseña de seguridad del terminal.
--> En caso de que le falte algún dato de los mencionados arriba, debe contactar con su soporte técnico de Addon Payments llamando al teléfono 914 353 028 (opción 2) o enviando un email al correo electrónico soporteaddonpayments.com.
ServicesContainer.ConfigureService(new GatewayConfig {
MerchantId = "addonnettest", // Identificador del comercio
AccountId = "api", // Subcuenta asociada a su identificador
SharedSecret = "secret", // Contraseña del terminal
ServiceUrl = "https://remote.sandbox.addonpayments.com/remote" // Se debe emplear una de las siguientes URL's:
// Sandbox URL para el entorno de pruebas - https://remote.sandbox.addonpayments.com/remote.
// Live URL para el entorno de producción - https://remote.addonpayments.com/remote.
});
2.2 Datos del cliente

Para crear el objeto "customer" necesitamos los siguientes parámetros:

  • Llamamos a la clase new Customer
  • Generamos un ID único para cada cliente llamando a la función Guid.NewGuid().ToString()
    • En caso de querer enviar un identificador personalizado, debemos enviar la siguiente información:
      • key = "Valor personalizado"
    • El identificador lo podemos utilizar posteriormente para editar el cliente que deseemos.
  • Title = Título (Señor/Señora)
  • FirstName = Nombre
  • LastName = Apellidos
  • DateOfBirth = Cumpleaños
  • CustomerPassword = En caso de que la hubiera, indicamos la contraseña del cliente
  • Email = Correo electrónico
  • HomePhone = Teléfono de casa
  • DeviceFingerPrint = En caso de que la hubiera, indicamos la huella digital
  • Fax = Fax
  • WorkPhone = Teléfono del trabajo
  • Comments = Comentarios
  • Company = Compañía
var customer = new Customer {
Key = Guid.NewGuid().ToString(),
Title = "Señor/Señora",
FirstName = "Nombre",
LastName = "Apellidos",
DateOfBirth = "Cumpleaños",
CustomerPassword = "Password",
Email = "Email",
HomePhone = "Teléfono",
DeviceFingerPrint = "Huella digital",
Fax = "Fax",
WorkPhone = "Teléfono del trabajo",
Comments = "Comentarios",
Company = "Compañía",
// Aquí va la sección '2.3 Dirección del cliente'
};
2.3 Dirección del cliente

Para crear el objeto "Address" necesitamos los siguientes parámetros:

  • Llamamos a la clase new Address
  • StreetAddress1 = Dirección de envío 1
  • StreetAddress2 = Dirección de envío 2
  • StreetAddress3 = Dirección de envío 3
  • City = Ciudad
  • Province = Provincia
  • PostalCode = Código postal
  • Country = ISO del país del cliente
Address = new Address {
StreetAddress1 = "Dirección 1",
StreetAddress2 = "Dirección 2",
StreetAddress3 = "Dirección 3",
City = "Ciudad",
Province = "Provincia",
PostalCode = "Código postal",
Country = "ISO del país (ES, GB, US...)",
}
2.4 Envío de la operación

Para enviar la operación debemos emplear el modelo de excepciones "try - catch". -->Para obtener más información sobre la excepción de "try - catch" puede ver la documentación oficial de .NET

  • Llamamos al objeto "customer" y ejecutamos la función de creación de usuario = customer.Create();
  • En caso de que queramos actualizar el usuario, debemos ejecutar la función = customer.SaveChanges();
try {
var response = customer.Create();
var payerRef = response.Key;

// Devolvemos la respuesta asociando los parámetros recibidos por el servidor a una variable
// para más información puede ver el controlador "AuthorizationApiController.cs"
return Ok(response.Key);

// Aconsejamos guardar los datos de la operacion en su base de datos en caso de querer gestionar de forma interna sus transacciones

} catch (ApiException exce) {
// Añada aquí su tratamiento de errores
return Ok(exce);
}

Para la respuesta realizamos un return Ok() de "exce".

Con esta función obtenemos el motivo de la denegación del servidor. Para ver todos los errores de denegación actualizados, puede comprobar nuestra documentación de Addon Payments.
